## 1. Core Doctrine

Memory is sovereign only when durable state lives in repo-visible artifacts or in subordinate systems that point back to repo truth.

Therefore:

- repo-ratified files are memory authority
- office-local and runtime surfaces are operational memory, not hidden law
- platform-native memory, thread history, and dashboard state are caches
- anything another agent or future session must reliably recover belongs in a file

## 2. Shell Memory Stack

1. **Law and direction**
   `AGENTS.md`, executive surfaces, program contract, canon, Sigma, playbooks, operators, and validated patterns.

2. **Operational state**
   handoffs, response receipts, backlog deltas, manifests, runtime snapshots, and office-local memory.

3. **Subordinate projections**
   exocortex records, ontology projections, connector state, and other externalized control-plane matter that may coordinate memory but cannot ratify it alone.

4. **Ephemeral cache**
   thread context, platform memory, temporary tool output, and unstored local notes.

## 3. Promotion And Decay

- Promote repeated operational wins upward into Sigma, canon, playbooks, operators, or validated patterns.
- Externalize unique decisions before thread loss; chat-only decisions do not count as durable memory.
- Let caches decay after their load-bearing state has been captured.
- Cull or archive stale operational memory once the durable lesson has been promoted.

## 4. Operator Obligations

- Keep the current objective, active constraints, and next action in files whenever work crosses sessions, agents, or harnesses.
- Prefer authoritative pointers over re-explaining history from scratch.
- Treat platform memory as convenience only; never assume portability or auditability.
- If an external system holds critical state, write a repo receipt or pointer.

## 5. Failure Modes

- hidden sovereign memory in chats, dashboards, or office-local notes
- accumulation without compaction or promotion
- external systems becoming the only place where operational truth can be recovered
- law, direction, evidence, and scratch state collapsing into one document
