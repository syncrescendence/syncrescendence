## 1. Core Doctrine

Context transition is inevitable and lossy. The shell does not try to avoid loss; it bounds loss and preserves fast re-entry.

The older unified-interface thesis remains correct, but in the successor shell the transition must land as lawful artifacts rather than chat-only summaries.

## 2. Modes

- **Continuation**: preserve the exact entry point for the next session or agent.
- **Culmination**: capture what changed, what was decided, and what deserves promotion.
- **Compression**: reduce token load while keeping the minimum state required to proceed.

The agent should choose the minimum valid combination. Most real transitions are hybrid.

## 3. Minimum Transition Packet

Every nontrivial transition should preserve:

- `state`: current objective and exact work position
- `constraints`: active decisions, assumptions, blockers, and no-go boundaries
- `pointers`: authority-bearing files, receipts, and inputs to reload
- `next`: the first concrete action after resumption
- `promotion candidates`: only when a stable lesson or doctrine candidate emerged

This packet may live in a handoff, response receipt, office-local memory update, backlog delta, or direct promotion artifact. If no durable artifact changed, the transition probably did not complete.

## 4. Trigger Rules

Run a transition when any of these becomes true:

- the session is ending or pausing with nontrivial work in flight
- context pressure, degraded recall, contradiction, or looping appears
- work is moving between harnesses, offices, or humans
- enough time will pass that re-entry cost becomes material
- a stable insight appeared that should outlive the thread

Do not wait for exhaustion. Compact at natural boundaries.

## 5. Shell Rules

- Prefer pointers to authoritative files over long narrative replay.
- Record owner, expected next step, and blocking dependency for cross-agent handoffs.
- Separate durable decisions from speculation and scratch space.
- Promote stable lessons upward; let transient narrative fall away after capture.
- Keep the artifact compact enough to reload quickly and specific enough to resume work without rediscovery.

## 6. Failure Modes

- summary without a next action
- handoff without authority pointers
- compression performed after key state was already lost
- culmination that names insights but never lands them in the proper lane
- continuation artifacts that silently rewrite law or intent
