- **role-title inflation**: Strategos / Synaptarch / Viceroy / Exegete and similar systems should be handled carefully and usually kept as pedigree or flavor, not operational burden
- **renaming cascades**: frequent successor-term churn should be treated as shell debt
- **mega-glossary bundling**: do not let live Rosetta collapse back into one file doing six jobs
- **mythic density without tiering**: poetic signal is valuable, but only when bounded by artifact law

## Successor-Shell Rule

The live Rosetta should contain only:

1. terms that are currently operationally load-bearing
2. terms whose definitions are required for lawful shell operation
3. terms whose absence would create active semantic debt

Everything else belongs in pedigree, assessment, playbooks, program, or future research.
