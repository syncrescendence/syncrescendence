# Oracle Patterns

## High-Value Patterns

### 1. High-context injection

Give Oracle the actual repo state, actual current problem, and actual anchor files.

### 2. Named anchor forcing

Force Oracle to engage specific files or artifacts instead of free-associating.

### 3. Ugly quote proof

Require rough, real verbatim snippets rather than polished paraphrase masquerading as proof.

### 4. Structural question framing

Ask “what is the architecture, tension, or blind spot?” not just “what’s the answer?”

### 5. Output-contract pressure

Specify sections, explicit deliverables, and what kinds of overreach are disallowed.

### 6. Assessment-after-landing

Every useful Oracle response gets evaluated in a repo-native assessment artifact before adoption.
