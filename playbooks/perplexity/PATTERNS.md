# Perplexity Patterns

## High-Value Patterns

### 1. Verification packet discipline

State the exact claim and why it matters.

### 2. Source-class constraints

Prefer primary and official sources when the question is about product or capability reality.

### 3. Counterexample pressure

Ask for what would disconfirm the thesis, not just what supports it.

### 4. Distinction checks

Use Perplexity to separate surfaces that are easy to conflate:
- API vs subscription
- browser surface vs model surface
- Google family sub-surfaces
- xAI API vs Grok web

### 5. Assessment after landing

Perplexity’s citations should become repo inputs, not automatic truth.
