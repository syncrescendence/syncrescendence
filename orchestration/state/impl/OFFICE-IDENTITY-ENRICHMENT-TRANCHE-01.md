- Enriched all primary office playbooks with:

  - fiduciary burden
  - native failure mode

- Enriched office READMEs so local spaces carry the same role distinctions as playbooks

## Design Rule

Identity language is allowed only when it improves:

- routing
- burden clarity
- escalation behavior
- anti-pattern avoidance
- memory discipline

Identity language is not allowed to become:

- a second canon
- decorative mythology with no operating consequence
- a reason to blur office boundaries

## Immediate Effect

The live shell now presents offices as burden-bearing institutions with differentiated grain, rather than as generic folders plus procedural notes.
