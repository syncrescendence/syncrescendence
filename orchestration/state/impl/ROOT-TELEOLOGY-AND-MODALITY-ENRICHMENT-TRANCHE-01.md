## Change Summary

The live root operator surfaces now carry:

- shell teleology
- office modalities
- work metabolism
- antifragile improvement criteria
- harness-specific reminders about why Claude and Gemini exist in the shell

This restores the predecessor shell's higher-fidelity orientation while keeping the live root partitioned by burden:

- `README.md` = shell purpose and modal structure
- `BOOT.md` = anti-drift session entry
- `WORK-LOOP.md` = work metabolism
- `INTER-OFFICE.md` = office modalities and routing burden
- `CONTINUOUS-IMPROVEMENT.md` = compaction, regulation, QA, automation
- `CLAUDE.md` / `GEMINI.md` = harness modality

## Design Rule

Teleology belongs at the root when it improves:

- orientation
- office differentiation
- routing quality
- developer happiness
- compaction discipline

Teleology does not belong at the root when it collapses constitutional law, executive intent, runtime facts, and harness-specific doctrine into one giant slab.
