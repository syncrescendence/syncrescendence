## 1. What The Repo Must Continue To Ratify

The repo must continue to ratify every artifact class that changes lawful meaning, directional burden, or institutional permission.

That includes:

- doctrine, constitutional law, shell rules, and executive intent
- surface teleology when it changes what a surface is for
- promotion, disposition, retirement, and migration decisions
- validated patterns, playbooks, operator rules, and office burdens
- authority-bearing registry definitions and any schema rule that would otherwise become hidden doctrine
- ratified interpretations of exocortex evidence and ontology projections

Repo law should stay compact. It does not need to hold every live receipt, but it must remain the only place where a transient fact becomes durable shell truth.

## 2. What The Exocortex Should Increasingly Coordinate Or Externalize

The exocortex should increasingly hold live operational burden that is real, queryable, and useful, but not sovereign.

It should coordinate or externalize:

- connector topology, auth dependencies, readiness state, and verification tranches
- execution receipts, polling outcomes, bounded automation state, and surface-local routing context
- external work queues, dashboards, SaaS-native operational scaffolds, and reversible control-plane state
- capture policies and evidence snapshots needed to support later repo ratification
- cross-surface synchronization tasks that are too dynamic or voluminous for repo residence

Exocortex expansion is good when it removes operational clutter from the repo. It is bad when it becomes the only place where policy, legitimacy, or direction can be discovered.

## 3. What The Ontology Should Project Rather Than Govern

The ontology should project typed legibility, not governance.

It should project:

- normalized entities for repo-ratified artifacts, exocortex snapshots, and their lineage
- typed relations across surfaces, offices, programs, connectors, receipts, and dispositions
- current status views, gap maps, dependency graphs, and tranche-readiness signals
- comparative views that expose conflicts between repo law and exocortex observation
- compact query surfaces over ratified meaning and subordinate operational state

It should not govern:

- doctrine
- priority
- surface legitimacy
- promotion
- semantic reinterpretation by schema convenience

Ontology value comes from projection discipline. If a graph node cannot point back to a repo artifact or subordinate snapshot, it should not be treated as shell truth.
