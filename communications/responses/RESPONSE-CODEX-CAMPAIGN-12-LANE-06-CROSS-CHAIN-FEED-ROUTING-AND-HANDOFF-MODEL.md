### 2. The cross-chain routing model

The routing model should be three-plane, not five separate intake stacks.

#### Plane A. Chain-owned curation

Each account may own its curated subscription manifest as source-of-truth for what that chain follows.
That is a curation authority, not an execution authority.

- Acumen account owns sensing-oriented feeds
- Coherence account owns framework, essay, and synthesis-oriented feeds
- Efficacy account owns operator, builder, and execution-oriented feeds
- Mastery account owns pedagogy, explanation, and curriculum-oriented feeds
- Transcendence account owns meta-pattern, civilizational, and long-horizon feeds

Those manifests should be captured and preserved per lane 03 and lane 04 logic, but they should not become five separate raw processing queues.

#### Plane B. Governed intake and triage

Acumen remains the default intake and triage authority for imported feed material.
Imported chain manifests widen the governed source constellation, but Acumen still decides what enters the active machine.

Default rule:

- raw feed candidates route into Acumen
- Acumen emits triage decisions, briefs, verification dossiers, repo-side assessments, and primary-source queues
- later chains consume those derivative surfaces by default

Exception rule:

- a later chain may receive a bounded primary-source queue or dossier only after Acumen or an Acumen-linked assessment has declared that the original form is still needed
- later chains may not self-constitute a raw candidate stream from their own subscriptions as an equal intake path

#### Plane C. Downstream chain consumption

| Chain | Default inbound artifact | Lawful role | Default stopping point |
| --- | --- | --- | --- |
| Acumen | raw candidates from imported manifests, triage packets, verification bridge state | sensing, qualification, escalation choice | triage decision, Dawn Brief, verification dossier, repo-side assessment, primary-source queue |
| Coherence | `VERIFIED-SIGNAL-BRIEF`, repo-side assessments, bounded primary-source queue, convergent promoted clusters | synthesis, framework extraction, pattern integration | synthesis memo, model, or framework dossier |
| Efficacy | Coherence synthesis, Acumen assessed signals with direct operational implications, selected primary-source tasks | operationalization, protocol choice, project routing | task packet, protocol, execution plan, or backlog item |
| Mastery | Coherence synthesis, Efficacy-tested procedures, verified exemplars, bounded primary-source teaching queue | curriculum formation, explanation, teaching sequence design | lesson, curriculum module, drill, or teaching note |
| Transcendence | cross-chain telemetry, Coherence synthesis, Efficacy outcomes, Mastery curricula, rare high-value primary-source escalations | meta-integration, phase reading, strategic review, governance correction | monthly meta-pattern report, quarterly review, or doctrine-pressure signal |

This yields the lawful default route:

`chain-owned feed manifest -> capture/import -> Acumen intake and triage -> assessed downstream artifacts -> later-chain specialization`

Not this:

`five accounts -> five independent raw inboxes -> five competing interpretations of the same source field`

### 3. Chain-specific stopping points

The stopping-point law should be explicit.

#### Acumen

Acumen stops when an item has been lawfully placed into one of four downstream states:

1. `Dawn Brief` inclusion for awareness-only items
2. verification dossier / portfolio admission for claims that need witness checking
3. repo-side assessment for post-verification classification
4. primary-source queue when compression is inadequate or ambiguity remains load-bearing

Acumen should not also try to do final synthesis, operational protocoling, curriculum design, and strategic meta-review in the same pass.

#### Coherence

Coherence should usually stop at synthesis.
Its job is to integrate assessed signal into models, comparisons, framework updates, or longer-form understanding.

It should not:

- reopen raw feed triage
- own verification intake
- silently mint operational commitments

#### Efficacy

Efficacy should stop at execution-shaping artifacts:

- task packets
- protocol updates
- backlog changes
- demonstration-domain plans

It should consume assessed or synthesized material, not adjudicate source truth from scratch unless a bounded primary-source task has already been assigned.

#### Mastery

Mastery should stop at transmission-ready structure:

- lesson modules
- syllabi
- explanation stacks
- practice loops

It should consume verified exemplars and synthesized understanding, not a river of raw feeds.

#### Transcendence

Transcendence should stop at meta-governance outputs:

- cross-chain pattern reports
- phase-transition reads
- strategic reviews
- doctrine-pressure and correction signals

It should be the last integrator, not another intake surface.

### 4. Highest-value handoff boundaries

The highest-value boundaries are the places where authority should change form.

1. `curated manifest -> Acumen registry admission`
   - this is where account-level curation becomes governed machine-readable intake
   - it is the key anti-sprawl gate
2. `Acumen triage decision -> verification / assessment path`
   - this separates awareness from truth-sensitive follow-up
   - it prevents every interesting item from becoming a research campaign
3. `repo-side assessment / primary-source queue -> Coherence`
   - Coherence should inherit assessed ambiguity and resolved facts, not raw noise
   - when deep reading is needed, it should pull from a bounded queue with explicit reason codes
4. `Coherence synthesis -> Efficacy and Mastery split`
   - Efficacy consumes what should change action
   - Mastery consumes what should change understanding and teaching
5. `cross-chain outputs + telemetry -> Transcendence`
   - Transcendence should integrate yield, failure modes, and repeated motifs across the other chains
   - this keeps meta-governance downstream of evidence instead of upstream of intake

### 5. Relationship to the current Acumen-led cycle

The current truthful state is that Acumen is the only real repo-native front-end intelligence lane.
So the immediate routing law should be asymmetric by design:

- all five accounts may contribute curated source constellations
- only Acumen should metabolize raw inbound feed flow by default
- later chains should first attach to Acumen outputs as consumers of products, assessments, and bounded source queues

That preserves sovereignty and keeps the next expansion lawful.
The five-account constellation widens what can be seen.
It does not yet justify five separate raw intake authorities.
