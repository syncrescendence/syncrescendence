## 1. Canon Admission Criteria

An artifact should enter `knowledge/canon/` only if it passes **all** of the following gates.

1. **Default-binding gate**: a competent agent should operate by this rule unless a more specific live instruction overrides it.
2. **Cross-surface gate**: it governs more than one immediate context, lane, or harness; if it matters only for one workflow, it is not canon.
3. **Durability gate**: the core claim should plausibly survive at least one year without depending on transient platform specifics.
4. **Compression gate**: it can be rendered compactly enough to be reloaded often; if it requires large annexes to make sense, the annex belongs in Sigma.
5. **Prescriptive gate**: it states governing distinctions, obligations, or prohibitions, not merely useful observations.
6. **Evidence gate**: it is supported by either repeated live operational proof or dual witness from lineage plus current-shell fit.
7. **Non-substitutability gate**: the artifact would lose force if reduced to a playbook, operator, validated pattern, or reference note.
8. **Supersession gate**: its relation to prior witnesses is explicit; canon cannot accept anonymous duplicates or silent rewrites.

Practical test:

- If the shell would repeatedly make bad default decisions without this artifact, promote toward canon.
- If the shell can still function without loading it on every pass, it is probably Sigma instead.

## 2. Sigma Admission Criteria

An artifact should enter `knowledge/sigma/` if it passes the Sigma gates and fails at least one canon gate for legitimate reasons.

### Sigma core gates

1. **Repeated-consult gate**: it is worth revisiting across multiple tasks or episodes, not just one migration decision.
2. **Secondary-doctrine gate**: it sharpens judgment, design, or reasoning, but does not need to bind every pass.
3. **Bounded-scope gate**: it applies to a domain, discipline, or doctrinal family rather than the whole shell by default.
4. **Moderate-durability gate**: its core survives beyond immediate news or tooling churn, even if examples change quarterly.
5. **Compaction-yield gate**: it contains distinctions worth preserving above raw source level.
6. **Non-executable gate**: its center of gravity is not deterministic mechanics; if it is mostly steps, scripts, or queue-state logic, it belongs in playbooks/operators/patterns.
7. **Non-pedigree gate**: it has present utility, not merely historical authenticity.

### Sigma/reference subtype gates

Within Sigma, `references/` should house artifacts that are:

- distilled from external or mixed-source research
- repeatedly useful but not yet localized into shell-native doctrine
- evidence-bearing support for future Sigma or canon promotion
- unsuitable as canon because they are still compendium-like, comparative, or source-proximate

Practical test:

- If the shell should consult it often for a class of problems, but not load it as default scripture, it belongs in Sigma.
- If it is still mainly evidence or compendium for later digestion, it belongs in `Sigma/references`.

## 4. Exclusion Rules

### Canon exclusions

The following do **not** belong in canon:

- raw source bundles, transcripts, notebooks, or bulky research sediment
- vendor catalogs, model/version matrices, dated tool comparisons, and platform price/performance snapshots
- deterministic workflow mechanics better represented as playbooks, operators, or validated patterns
- office-local memory, harness-local heuristics, inbox state, or communications artifacts
- historical scripture or topology-bearing originals whose present force is ancestral rather than binding
- unresolved worldview or metaphysical material that does not presently govern shell operation

### Sigma exclusions

The following do **not** belong in Sigma:

- raw intake that has not yet been compacted or qualified
- ephemeral troubleshooting notes with short half-life
- mostly executable mechanics, scripts, lifecycle procedures, or handoff templates
- nostalgia artifacts preserved for authenticity but not live use
- duplicate canon restatements with no added annex depth
- large undifferentiated corpora whose only merit is volume

### References subtype exclusions

The following do **not** belong in `Sigma/references`:

- shell-binding mandates phrased as if already ratified
- personal or office scratch memory
- unprocessed source drops
- purely archival originals better held in pedigree

## 6. Promotion Rule of Thumb

Use this sequence during actual migration:

1. Ask whether the artifact is **binding law**, **secondary doctrine**, **reference substrate**, **executable mechanics**, **raw intake**, or **pedigree**.
2. If omission would break default shell judgment, test for **canon**.
3. If omission would not break default judgment but would weaken repeated reasoning in a bounded area, test for **Sigma**.
4. If the artifact is still mainly compendium or evidence, place it in **`Sigma/references`**.
5. If the artifact is mostly steps, state transitions, or execution mechanics, route it to **playbooks/operators/validated-patterns** instead.
6. If the artifact's main value is ancestry, preserve it in **pedigree** with receipt rather than forcing it live.

## 7. Top Failure Modes If Stratification Is Not Enforced

1. **Canon inflation**: every good artifact gets promoted upward, canon becomes too large to reload, and agents stop actually obeying it.
2. **Sigma erasure**: the shell loses a lawful home for secondary doctrine, so material jumps directly from feedstock to canon or leaks sideways into playbooks.
3. **Reference-law confusion**: imported `neocorpus` entries begin acting as live mandates without local ratification.
4. **05-SIGMA relapse**: mechanics, operator logic, proverbs, research syntheses, and cautionary tales collapse back into one miscellaneous bucket.
5. **Pedigree leakage**: old scripture and topology-specific artifacts regain authority by adjacency rather than adjudication.
6. **Volatility contamination**: dated platform or tooling specifics quietly enter canon and force unnecessary rewrites of supposedly durable law.
7. **Duplicate doctrine divergence**: the same topic appears in canon, Sigma, playbooks, and references with no declared supersession path.
8. **Compaction stall**: without a real Sigma rung, artifacts either stay as bulky references forever or get over-compressed into brittle canon.
