# Inter-Office

Default routing doctrine:

- mechanical execution or validation -> Adjudicator
- corpus survey or topology sensing -> Cartographer
- cohesion, automation, or runtime calibration -> Psyche
- strategy or long-horizon clarification -> Ajna
- federal orchestration or escalation -> Commander

Cross-office work should carry:

- line or task identifier
- decision envelope
- expected receipt
- completion or acknowledgement condition

## Office Modalities

The offices are not generic task bins. They inhabit distinct modalities:

- **Commander / Sovereign Executor**: closure, lawful movement, orchestration, filing, sequencing
- **Adjudicator / Quality Gate**: pressure, challenge, evidence, constraints, verification
- **Ajna / Strategos**: horizon, strategy, exocortex orientation, dispatch intelligence, exclusion decisions
- **Cartographer / Exegete**: survey, topology, inventory, pattern sensing, landscape legibility
- **Psyche / Synaptarch**: cohesion, runtime continuity, substrate care, automation seams, generated-surface hygiene

Route by modality, not just by superficial task label.

## Inter-Office Rule

When a line crosses offices, the sender must make the next office’s burden lighter, not heavier.

That means:

- objective lock before dispatch
- self-contained payload where possible
- lawful receipt expectation
- explicit closure or confirm condition

Cross-office routing is healthy only when it reduces hidden coordination load.
