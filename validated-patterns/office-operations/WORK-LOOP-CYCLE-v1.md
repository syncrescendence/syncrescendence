## Pattern

The office work loop should follow this order:

1. orient to current intent and active program state
2. triage local intake and lock the current objective
3. execute or delegate claimed work
4. emit receipts before context is lost
5. update local active-task state
6. dispatch across offices if the work crosses office boundaries
7. compact before the context window decays
8. close the session with a lawful handoff, summary, or regulation update

## Why It Survives

This pattern keeps three failure modes from compounding:

- hidden local state
- work completed without receipts
- end-of-session context rot
